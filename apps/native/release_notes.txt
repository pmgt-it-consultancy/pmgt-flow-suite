## What's New

- Cart taps feel instant — adding items, adjusting quantities, toggling dine-in/takeout, and removing items update immediately without waiting for the server
- Rapid +/- presses on cart items are coalesced into a single update instead of firing one per tap
- Home screen redraws less often during busy service — clock, order counts, and revenue tiles no longer re-render on unrelated activity
- Printer status changes (reconnects, Bluetooth polling) no longer cause the order entry or checkout screen to refresh
- Modifier groups load only when you tap a product — the app no longer pre-loads every modifier in the store on screen open
- Home screen "Active Orders" pulls roughly half the data it used to, so the list populates faster after each order change
- Order entry product grid scrolls smoother on large menus

## Notes

- Requires a one-time backend migration (`backfillOrderDenorm`) to populate denormalized fields on existing open orders; runs automatically during deploy
