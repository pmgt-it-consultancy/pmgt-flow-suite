## What's New

- Refund paid orders — staff can now refund individual items from paid orders; the system voids the original and creates a new order with the remaining items, auto-settled
- Refund method selection — choose cash or card/e-wallet when processing a refund
- Refunded order tracking — replacement orders show a "Refunded" badge so cashiers can identify and reprint receipts
- Refund available everywhere — refund from Order History, Takeout completed orders, or the web admin panel
- Item-level service type — toggle individual items between Dine-in and Takeout within the same order, with exception badges on receipts and kitchen tickets
- Reports accuracy — refund voids use the actual refund amount instead of the full order total, preventing double-counting

## Notes

- Refunds require manager PIN approval, same as voids
- The original order is voided; a new order with a fresh order number is created for the remaining items
- Discounts from the original order are reapplied to the replacement order automatically
