## What's New

- Offline-first sync with WatermelonDB — all catalog reads, order writes, and mutations now go through local SQLite first, then sync to Convex in the background.
- Quantity +/- buttons now debounced (300ms) so rapid taps don't fire individual mutations.
- Takeout status changes (pending → preparing → ready → completed) now re-render immediately without navigating away.
- Sync layer handles push deletions, pull deduplication, and existing-row conflict resolution.
- Batch sync performance — existing-row detection during pull uses a single SQL query per table instead of N individual lookups.

## Notes

- Requires backend deploy of the updated /sync/pull and /sync/push endpoints.
- First launch after upgrade triggers a full re-pull to refresh local catalog data.
