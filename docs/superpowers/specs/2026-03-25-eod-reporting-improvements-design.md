# End-of-Day Reporting Improvements Design

**Date:** 2026-03-25
**Status:** Approved

## Overview

Improve end-of-day reporting across the POS native app and web admin:
1. Restructure the POS Day Closing screen with item breakdown and better date navigation
2. Remove unused batch receipt reprint functionality
3. Add PDF report generation to the web admin for reconciliation

## POS Native App — Restructured Day Closing Screen

### Screen Layout (top to bottom)

1. **Header bar** — "Day Closing" title + back button + refresh/generate report button
2. **Date navigation bar** — Full-width, prominent:
   - Left arrow | Date display (e.g. "Wed, Mar 25, 2026") | Right arrow
   - Tapping date text opens calendar picker
   - Right arrow disabled when date = today
   - Light blue background (`#EFF6FF`) so it reads as interactive
3. **Scrollable content** (single `ScrollView`, NO nested scrollable containers):
   - **Z-Report Summary card** — Financial stats: gross/net sales, transactions, cash, card, discounts, voids, VAT, avg ticket. Print button removed from this card.
   - **Item Breakdown section** — Card with "Items Sold" header and product count. Plain mapped list (NOT FlatList) showing: Product Name, Qty, Amount. Voided items shown with voided qty/amount in red. Sorted by quantity descending.
4. **Sticky footer** — Full-width "Print Z-Report" button (primary, `#0D87E1`). Disabled if no report generated.

### Scrolling Constraint

The screen uses a single `ScrollView` as the only scrollable container. The item breakdown is rendered as a plain `.map()` — never a `FlatList` or `SectionList` nested inside — to avoid React Native's nested virtualized list warnings and scrolling issues.

## Web Admin — PDF Report Generation

### UI Change

Add "Download PDF" button to the reports page, next to "Mark as Printed". Uses download icon from lucide-react. Enabled only when a report exists for the selected date.

### PDF Document Structure (`@react-pdf/renderer`)

1. **Store Header** — Store name (large, bold), address, TIN, contact info
2. **Report Title** — "Daily Sales Report" + date + "Generated by [name] on [datetime]"
3. **Financial Summary** — Two-column layout:
   - Left: Gross Sales, Net Sales, Transactions, Average Ticket
   - Right: Vatable Sales, VAT Amount, VAT-Exempt, Non-VAT
4. **Discounts & Voids** — Senior, PWD, Promo, Manual totals + Void count/amount
5. **Payment Methods** — Cash, Card/E-Wallet totals
6. **Product Sales Breakdown** — Table: Product Name, Category, Qty Sold, Gross Amount, Voided Qty, Voided Amount
7. **Sales by Category** — Category name, product count, total qty, total amount
8. **Hourly Breakdown** — Table: Hour, Transactions, Net Sales (skip zero-activity hours)
9. **Footer** — "This is a system-generated report" + print timestamp

### New Files

- `apps/web/src/app/(admin)/reports/_components/ReportPdfDocument.tsx` — React-PDF component
- `apps/web/src/app/(admin)/reports/_components/DownloadPdfButton.tsx` — Button triggering PDF generation and download
- `apps/web/src/app/(admin)/reports/_components/index.ts` — Barrel export

### New Dependency

- `@react-pdf/renderer`

### Data Sources

All queries already exist — no backend changes needed:
- `getDailyReport` — Financial summary
- `getDailyProductSales` — Product breakdown
- `getHourlySales` — Hourly data
- `getCategorySales` — Category aggregates
- `stores.get` — Store info for header

## Approach

Approach 1 (Incremental Refactor) — modify existing files in-place. The Day Closing screen is unused so a full rewrite is safe. Backend queries already provide all needed data. The work is primarily UI: restructuring one native screen, updating the thermal formatter, and adding a PDF export component to the web.
